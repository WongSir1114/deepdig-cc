## 一、产品定义

### 1.1 这是什么

**深度挖掘 · CC** 是一个 Obsidian 社区插件。用户在 Obsidian 侧边栏里输入"深挖宁德时代 300750"，插件在后台召唤 Claude Code CLI（Opus 4 模型）做投研分析，流式返回对话式分析结果，同时静默将知识卡片写入 Obsidian vault。

### 1.2 交付物

| 交付物 | 说明 |
|------|------|
| 插件本身 | `deepdig-cc` — Obsidian 插件（main.js + manifest.json + styles.css） |
| Vault 模板 | 目录结构（`1-原子笔记/实体/` 等）+ CLAUDE.md（214行投研规则） |
| 一键安装器 | Windows `.exe` / macOS `.pkg` — 含 Node.js + CC CLI + 插件 + 模板 |
| 安装指南 | 图文网页 — "3 步装好深度挖掘" |

### 1.3 目标用户

- 会用 Obsidian 写投资笔记的散户投资者
- 对技术不太熟悉——不想碰终端、npm、环境变量
- 投研知识深度需求超过 ChatGPT 免费版能提供的

## 二、系统要求

### 2.1 操作系统

| 系统 | 版本 | 状态 |
|------|------|:--:|
| Windows | 10 / 11（64 位） | ✅ 主要支持 |
| macOS | 12.0 Monterey+（Intel + Apple Silicon） | ✅ 支持 |
| Linux | Ubuntu 22.04+ / Debian 12+ | ⚠️ 仅脚本安装，无 .exe 安装器 |

### 2.2 硬件配置

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 内存 | 4GB | 8GB+ |
| 硬盘 | 1GB 可用空间 | 5GB+（vault 持续增长） |
| 网络 | 宽带（CC 调用 Anthropic API） | 稳定宽带 |
| CPU | 双核 | 四核+ |

> **注意**：推理计算在 Anthropic 云端完成，用户电脑不跑模型。硬件要求主要是运行 Obsidian + Node.js。

### 2.3 依赖软件

| 软件 | 必须？ | 说明 |
|------|:--:|------|
| Obsidian | ✅ | v1.5.0+。插件的宿主 |
| Node.js | ✅ | v18+。CC CLI 的运行环境 |
| Claude Code CLI | ✅ | `@anthropic-ai/claude-code`。推理引擎 |
| Anthropic API Key | ✅ | 用户自己的 Key。CC 按量计费，一次深挖约 $0.1-0.5 |
| Git | ❌ | 非必须。仅管道脚本拉取财务数据时用到 |

> **关键设计决策**：API Key 由用户自行获取和管理。插件不代理 API 调用、不接触用户 Key、不承担推理成本。用户直接向 Anthropic 付费——我们不做中间商。

## 三、安装流程

### 3.1 核心原则

**用户最多做 3 步。** 每一步都可能流失——每一步都必须是"双击 → 下一步"式的简单操作。

### 3.2 三条安装路径

用户打开安装指南网页（如 `deepdig.beaver-cloud.com/setup`），页面自动检测用户当前有什么，展示对应的安装路径：

```

│  第 1 步：下载安装器                                  │
│  ┌─────────────────────────────────────────────┐    │

│  └─────────────────────────────────────────────┘    │

│  第 2 步：双击运行                                    │
│  安装器自动完成：                                     │
│  · 安装 Node.js（便携版，不污染系统）                  │
│  · 安装 Claude Code CLI                             │
│  · 引导安装 Obsidian（如未安装·打开官网下载页）         │
│  · 创建 DeepDig vault + 初始化目录结构               │
```

**安装器内部技术方案**：

| 平台 | 打包方式 | 内含 |
|------|---------|------|
| Windows | NSIS 或 Inno Setup 打包 `.exe` | Node.js portable + CC CLI + vault 模板 + 插件文件 |
| macOS | Shell 脚本 + Platypus 打包 `.app` | 同上（macOS 版 Node.js） |
| Linux | 自解压 `.sh` 脚本 | 同上 |

安装器大约 60MB（主要是 Node.js portable）。

**安装器运行逻辑**：

```
启动
├─ 1. 系统检测
│   ├─ OS 版本 + 架构（32/64位 / Intel/ARM）
│   ├─ 是否已有 Node.js → 有则跳过安装
│   ├─ 是否已有 Obsidian → 无则打开下载页
│   └─ 是否已有 CC CLI → 有则跳过安装

├─ 2. 安装 Node.js（如无）
│   └─ 解压 bundled Node.js portable → 设置 PATH
│
├─ 3. 安装 CC CLI
│   └─ npm install -g @anthropic-ai/claude-code
│
├─ 4. Obsidian 对接
│   ├─ 检测到 Obsidian → 检测现有 vault 列表
│   │   ├─ 有 vault → 让用户选择：①在已有 vault 中初始化 ②新建 vault
│   │   └─ 无 vault → 自动创建"深度挖掘"vault
│   ├─ 检测不到 Obsidian → 打开 obsidian.md/download
│   │   └─ 等待用户安装完成 → 回到 vault 对接
│   └─ 在目标 vault 中：
│       ├─ 创建目录：1-原子笔记/实体/、概念/、关键词/、关系/、投资思想/
│       ├─ 创建目录：2-MOC/、报告输出/
│       ├─ 写入 CLAUDE.md（214行·投研规则）
│       ├─ 复制插件文件到 .obsidian/plugins/deepdig-cc/
│       └─ 如有已存在的 CLAUDE.md → 备份到 _claude-versions/
│
├─ 5. API Key 配置
│   ├─ 弹出输入框 + 图文获取指引
│   ├─ 用户粘贴 → 写入系统环境变量 ANTHROPIC_API_KEY
│   ├─ 用户跳过 → 记录"待配置"，首次使用插件时再引导
│   └─ 验证：claude --version → 确认 Key 有效
│
└─ 6. 完成
    └─ 提示打开 Obsidian → 左侧 ribbon 图标 → 开始使用
```

## 四、分发渠道

### 4.1 渠道策略

| 阶段 | 渠道 | 定位 | 时效 |
|------|------|------|:--:|
| **首发** | 自己网站 `deepdig.beaver-cloud.com` | 安装器下载 + 安装指南 + 文档 | 立即可做 |
| **首发** | GitHub Releases | 插件文件 + 安装器 + 源码 | 立即可做 |
| **上架后** | Obsidian 社区插件商店 | 主要分发渠道。用户一键安装 | 提交审核 ~2周 |
| **长期** | 两者并行 | 商店 = 获客。网站 = 完整安装体验 | 长期 |

### 4.2 Obsidian 商店上架须知

**硬性要求**（来自 Obsidian 插件政策）：

| 要求 | 说明 | 我们的情况 |
|------|------|------|
| 完全免费 | 插件代码中不能有任何付费墙、功能限制、广告 | ✅ 可以做到 |
| 开源 | 需提供公开仓库（GitHub） | ⚠️ 需要清理敏感信息后开源 |
| 无外部付费依赖 | 插件本身不强制用户付费 | ✅ CC 是外部工具，非插件收费 |
| 隐私 | 不收集用户数据、不上传 vault 内容 | ✅ 纯本地运行 |
| 仅桌面端 | `isDesktopOnly: true` → 不合规移动端 | ✅ 我们就是桌面端 |

**提交清单**：

```
□ GitHub 公开仓库（含 README + LICENSE + 截图）
□ manifest.json 完整（id / name / version / minAppVersion / description / author / isDesktopOnly）
□ 代码无 obfuscation
□ 代码不加载远程资源（本地运行）
□ 代码不收集用户数据
□ 至少 1 张截图（聊天面板界面）
□ 英文 README（Obsidian 商店要求）
```

### 4.3 自己网站分发的额外优势

## 五、API Key 策略

### 5.1 为什么让用户自带 Key

| 原因 | 说明 |
|------|------|
| 成本归属 | 推理费由 Anthropic 直接向用户收取。我们不做中间商，不承担 API 成本 |
| 插件商店合规 | Obsidian 商店要求插件完全免费。如果插件收费或强制订阅 → 上架被拒 |
| 架构简单 | 不需要服务器做 API Key 代理。不需要处理计费、用量限制、欠费 |
| 隐私 | 用户 Key 存在本地。我们不接触、不存储、不中转 |

### 5.2 用户获取 Key 的引导

插件内置一个"配置向导" Modal：

```
┌──────────────────────────────────────────────┐

│                                               │

│                                               │
│  1. 点击下方按钮打开 Anthropic 控制台          │
│     [打开 console.anthropic.com]              │
│                                               │
│  2. 注册或登录 → API Keys → Create Key        │
│                                               │
│  3. 复制 Key 粘贴到下方：                      │
│  ┌───────────────────────────────────────┐   │
│  │ sk-ant-api03-...                      │   │
│  └───────────────────────────────────────┘   │
│                                               │
│  💡 新用户有 $5 免费额度                      │
│  💰 一次深度分析约消耗 $0.1-0.5               │
│  🔒 Key 只存在你的电脑上，不上传               │
│                                               │
│  [稍后配置]  [保存并验证]                      │
└──────────────────────────────────────────────┘
```

点击"保存并验证"后，插件内部执行：

```
1. 写入系统环境变量 ANTHROPIC_API_KEY
2. 执行 claude --version → 验证 Key 是否有效
3. 有效 → 关闭 Modal → 正常使用
4. 无效 → 显示错误提示 → 重新输入
```

### 5.3 Key 的存储位置

| 平台 | 存储位置 |
|------|---------|
| Windows | 系统环境变量 `ANTHROPIC_API_KEY`（安装器写入）或插件本地 data.json |
| macOS | `~/.zshrc` 或 `~/.bash_profile` 追加 export 行（安装器写入） |
| Linux | 同上 |

> **备选方案**：如果用户不想设系统环境变量，插件允许在设置面板中直接填入 Key，存于插件本地 data.json（Obsidian 的 `loadData/saveData` 机制）。CC spawn 时通过 `env` 参数传递给子进程。

## 六、首次使用体验

### 6.1 插件首次启动

插件被 Obsidian 加载后，做的第一件事：

```
插件 onload()
├─ 1. 检测 CC CLI 是否可用
│   ├─ 执行 claude --version
│   ├─ 失败 → 设置面板显示"CC 未安装。运行安装器或手动安装。"
│   └─ 成功 → ✅
│
├─ 2. 检测 vault 是否已初始化
│   ├─ 检查 vault 根目录是否有 CLAUDE.md
│   ├─ 有 → ✅
│   └─ 没有 → 弹出提示："是否初始化投研知识库？"
│       → [是] → 创建目录 + 写入 CLAUDE.md
│       → [否] → 跳过（用户可能在非投研 vault 中使用）
│
├─ 3. 检测 API Key 是否已配置
│   ├─ 检查环境变量 ANTHROPIC_API_KEY 或插件存储的 Key
│   ├─ 有 → ✅
│   └─ 没有 → 首次发送消息时弹出配置向导（见第五章）
│
├─ 4. 检查 CLAUDE.md 版本
│   ├─ 读取 vault 中的 CLAUDE.md → 提取 frontmatter 中的 version 字段
│   ├─ 与插件内置的最新版本号对比
│   ├─ vault 中的版本更旧 → 通知："投研规则有更新。是否同步？"
│   │   → [是] → 备份旧版 → 写入新版
│   │   → [否] → 保留旧版
│   └─ 版本一致 → ✅
│
└─ 5. 正常启动 → 聊天面板就绪
```

## 七、更新机制

### 7.1 各组件更新方式

| 组件 | 更新方式 | 频率 | 自动？ |
|------|---------|------|:--:|
| 插件本体 | Obsidian 商店自动推送 | 每次迭代 | ✅ 自动 |
| CC CLI | 插件检测版本 → 提示用户一键更新 | 跟随 Anthropic 发布 | ❌ 提示 |
| CLAUDE.md 规则 | 插件检测 → 弹出同步提示 | 开发者迭代后 | ❌ 提示 |
| Vault 模板 | 插件内置最新模板 → 新 vault 用新版 | 随插件发布 | ✅ 新用户自动 |

### 7.2 CLAUDE.md 规则同步

这是版本保护机制的一部分。当开发者更新了 CLAUDE.md（如增强子卡生成、修复回复格式），新版随插件发布。用户端处理：

```
插件 onload 检测到 vault 中 CLAUDE.md version < 插件内置 version
├─ 弹出通知："投研规则有新版本（v12.0 → v13.0）"
├─ [查看变更] → 打开 changelog
├─ [立即同步] →
│   ├─ 备份当前 CLAUDE.md → _claude-versions/{日期}-{序号}-自动备份.md
│   ├─ 写入新版 CLAUDE.md
│   └─ 告知用户："已更新。旧版已备份。"
└─ [暂不更新] → 关闭通知（下次启动再提示）
```

> **重要**：规则同步**不覆盖**用户手动修改的 CLAUDE.md。如果检测到用户修改过（mtime 晚于插件内置版本的发布时间 + 行数差异 > 5行），弹出更强的提示："检测到你自定义了投研规则。同步会覆盖你的修改（旧版会备份）。是否继续？"

### 7.3 CC CLI 更新

```
插件启动时 → 执行 claude --version → 获取当前版本号
  → 与上次记录对比
  → 检测到 Anthropic 有新版本发布（通过 GitHub API 查询 @anthropic-ai/claude-code 最新版）
  → 设置面板显示："CC CLI 有新版本（1.x → 1.y）[一键更新]"
  → 点击后：npm install -g @anthropic-ai/claude-code
  → 输出更新日志
```

## 八、许可证与激活（未来可选）

### 8.1 当前阶段：不设许可证

对于 Obsidian 商店上架，插件**不包含**任何许可证检查或付费墙。所有功能完全免费。

### 8.2 未来可能的演进

如果未来需要收费（如自建推理代理、高级功能），可选路径：

| 路径 | 说明 | 适用场景 |
|------|------|---------|
| 免费插件 + 付费服务 | 插件永远免费。高级功能通过服务器端 API 提供（如基金持仓分析、产业链图谱） | 不影响商店上架 |
| 自分发付费版 | 从商店下架或不分发商店版。通过自己网站分发含许可证的版本 | 需要强付费时 |
| 双版本 | 商店版 = 基础功能。网站下载的安装器 = 高级功能含许可证 | 最大灵活性 |

**当前建议**：保持纯免费。用免费插件获取用户基数 → 引导到其他变现渠道（付费报告、定制分析、企业版）。

这套体系在插件分发中要完整复制。安装器初始化 vault 时，连备份目录结构一起创建。

### 9.2 插件端需要新增的保护

| 保护 | 说明 |
|------|------|
| CC 进程超时 | 单次 CC 运行最长 30 turns。超过 → 自动 kill。防止死循环烧 API 费 |
| 异常恢复 | CC 进程崩溃 → 显示友好提示而非报错堆栈 |
| Vault 写入保护 | 插件不在 vault 外写入文件。CC 的 Write 权限受 `--allowedTools` 限制 |
| 用户数据不上传 | 代码中无任何网络请求（除可选的版本检查 + CLAUDE.md 同步） |

## 十三、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| Obsidian 商店审核不通过 | 最大获客渠道丢失 | ①提交前逐项核对政策 ②如果被拒 → 自分发 + 重新提交 |
| 用户获取 API Key 太困难 | 安装流失率高 | ①配置向导图文并茂 ②提供"跳过·稍后配置" ③考虑未来代理 API 调用 |
| Node.js 安装失败（权限问题） | 用户无法使用 | ①bundled portable 版避免系统权限 ②安装器以用户权限运行 |
| CC CLI 大版本更新不兼容 | 插件 spawn 失败 | ①锁 CC CLI 版本范围 ②及时跟进测试 |
| Anthropic 封禁中国 IP（如果用户代理翻墙用） | 部分用户无法使用 | 当前无解。标注在系统要求中："需要能访问 api.anthropic.com" |
| Obsidian 商店要求开源 | 代码公开 | ①清理敏感信息 ②MIT 许可证 ③接受社区贡献 |
